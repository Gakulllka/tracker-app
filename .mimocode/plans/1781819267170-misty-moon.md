# План: Массовые операции + Контекстное меню

## Контекст

Delta Tracker — трекер задач с таблицей (14 статусов, 5 приоритетов), беклогом, дашбордом, презентациями. Текущее состояние:
- Таблица поддерживает выбор ОДНОЙ строки (`selectedRowId: string | null`)
- Нет чекбоксов, нет мультивыбора
- Нет контекстного меню (правый клик)
- Компонент `DropdownMenu` из shadcn/ui уже используется в проекте

**Файл:** `src/components/views/table-view.tsx` (или новый `src/components/bulk-actions-bar.tsx`)

Горизонтальная панель, появляется при `selectedTaskIds.size > 0`:
```
[✓ Выбрано: 5] [Статус ▾] [Приоритет ▾] [Перенести ▾] [📦 Беклог] [🗑 Удалить] [Сбросить]
```

- **Статус** — dropdown со всеми статусами, при выборе → `bulkUpdateTasks(month, ids, "status", value)`
- **Приоритет** — dropdown со всеми приоритетами
- **Перенести** — dropdown с 12 месяцами, при выборе → `moveTasks` для каждой задачи
- **Беклог** — `moveToBacklog` для каждой задачи
- **Удалить** — `deleteTask` для каждой задачи (с undo snapshot)
- **Сбросить** — `clearSelection()`

## ФУНКЦИОНАЛ 2: Контекстное меню

### 2.1 Компонент контекстного меню

**Файл:** Новый `src/components/task-context-menu.tsx`

Используем shadcn `ContextMenu` (уже есть в `src/components/ui/context-menu.tsx`):

```tsx
<ContextMenu>
  <ContextMenuTrigger>{children}</ContextMenuTrigger>
  <ContextMenuContent>
    <ContextMenuItem>Перейти к статусу →</ContextMenuItem>
    <ContextMenuSub>
      <ContextMenuSubTrigger>Статус</ContextMenuSubTrigger>
      <ContextMenuSubContent>
        {Object.values(STATUSES).map(s => <ContextMenuItem>{s}</ContextMenuItem>)}
      </ContextMenuSubContent>
    </ContextMenuSub>
    <ContextMenuSub>
      <ContextMenuSubTrigger>Приоритет</ContextMenuSubTrigger>
      <ContextMenuSubContent>
        {Object.values(PRIORITIES).map(p => <ContextMenuItem>{p}</ContextMenuItem>)}
      </ContextMenuSubContent>
    </ContextMenuSub>
    <ContextMenuSeparator />
    <ContextMenuItem>Перенести в месяц →</ContextMenuItem>
    <ContextMenuItem>📦 В беклог</ContextMenuItem>
    <ContextMenuItem>📋 Дублировать</ContextMenuItem>
    <ContextMenuSeparator />
    <ContextMenuItem className="text-destructive">🗑 Удалить</ContextMenuItem>
  </ContextMenuContent>
</ContextMenu>
```

### 2.2 Пункты контекстного меню

| Пункт | Действие |
|-------|----------|
| **Статус →** | Sub-menu со всеми статусами, клик → `updateTask(month, id, "status", value)` |
| **Приоритет →** | Sub-menu со всеми приоритетами, клик → `updateTask(month, id, "priority", value)` |
| **Перенести в месяц →** | Sub-menu с 12 месяцами, клик → `moveTasks(id, fromMonth, toMonth)` |
| **В беклог** | `moveToBacklog(month, id)` |
| **Дублировать** | Копия задачи с новым id, добавляется в тот же месяц |
| **Удалить** | `deleteTask(month, id)` |

### 2.3 Интеграция в таблицу

**Файл:** `src/components/views/table-view.tsx`

Обернуть `<TableRow>` в `<ContextMenu><ContextMenuTrigger>`:

```tsx
<ContextMenu key={task.id}>
  <ContextMenuTrigger asChild>
    <TableRow onClick={() => setSelectedRowId(task.id)} className={...}>
      {/* ... все ячейки ... */}
    </TableRow>
  </ContextMenuTrigger>
  <TaskContextMenu task={task} month={month} ... />
</ContextMenu>
```

### 2.4 Дублирование задачи

Добавить action в store (`src/lib/store.ts`):
```ts
duplicateTask: (month: number, taskId: string) => void;
```

Реализация: найти задачу по id, создать копию с `crypto.randomUUID()`, добавить в тот же месяц.

## ВЕРИФИКАЦИЯ

1. `npm run build` — проверка сборки
2. `npm run lint` — проверка линтера
3. Ручная проверка:
   - ПКМ по задаче → появляется контекстное меню
   - Статус/приоритет меняются через контекстное меню
   - Дублирование задачи работает
   - Выбор нескольких чекбоксов → появляется панель массовых действий
   - Массовая смена статуса/приоритета
   - Массовый перенос в другой месяц
   - Массовое удаление
   - Ctrl+Z отменяет массовое действие
