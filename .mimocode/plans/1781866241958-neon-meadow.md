# Исправление запуска сайта

## Проблема 1: ~100 ГБ виртуальной памяти

Это **не диск**, а виртуальная память (page file) Windows. Next.js 16 + Turbopack потребляет огромное количество RAM. При закрытии консоли память освобождается — это нормально.

**Решение:** очистить кеш `.next/` перед запуском.

## Проблема 2: Сайт не загружается на localhost

Причина: `src/proxy.ts` — файл мог переместиться некорректно после переименования. Дополнительно Prisma модель `PlanfactTask` отсутствует в schema (причина крашей при использовании серверных действий).

## План действий

### Шаг 1: Очистка кеша Turbopack
```bash
rm -rf .next
```

### Шаг 2: Исправление proxy.ts
Удалить `src/proxy.ts`, создать `proxy.ts` в **корне проекта**. Конвенция `proxy` в Next.js 16 может требовать расположение в корне (как `middleware.ts`). Если не поможет — попробуем вернуть `middleware.ts` в корень.

### Шаг 3: Запуск и проверка
```bash
npm run dev
```
- Открыть http://localhost:3000
- Проверить что страница загружается без ошибок

### Шаг 4 (если не помогло): Откат proxy → middleware
Если `proxy.ts` не распознаётся:
- Удалить `proxy.ts`
- Создать `middleware.ts` в корне с функцией `middleware` (старый формат работает даже с предупреждением)

## Файлы
- Удалить: `src/proxy.ts`
- Создать: `proxy.ts` (корень проекта)
- Запасной вариант: `middleware.ts` (корень проекта)

## Верификация
1. `npm run dev` — нет критических ошибок
2. http://localhost:3000 — страница загружается
