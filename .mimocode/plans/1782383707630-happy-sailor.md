# План: пример идеальной презентации ежемесячного отчёта

## Что делаем

Создаём standalone HTML-файл `example-monthly-report.html` — готовый пример презентации ежемесячного отчёта по задачам, который можно открыть в браузере. Файл полностью самодостаточный (inline CSS + data), не зависит от приложения.

## Анализ текущей презентации — что не так

Текущая генерация (`slides.ts` + `presentation-renderer.tsx`) создаёт 7 слайдов:

| Слайд | Проблема |
|-------|----------|
| Title | Норм, но пустовато — только 3 цифры |
| KPI | 4 карточки с эмодзи — визуально плоско, нет сравнения с прошлым |
| Статусы | Горизонтальные полоски —.ok, но нет доли от общего |
| Завершённые задачи | Сетка 3×3 карточек — перегружено, мелкий текст |
| Задачи в работе | Аналогично — карточки сливаются |
| Таблица | 15 строк — невозможно читать на слайде, убивает презентацию |
| Итоги | 4 блока с маркерами — тексты generic, не привязаны к цифрам |

**Главные проблемы:**
1. Нет **визуального разнообразия** — все слайды центрированы с одинаковым layout
2. Нет **сравнения** (прошлый месяц, план/факт динамика)
3. **Таблица** не для презентаций — нужна визуализация
4. Нет **графиков** (хотя бы прогресс-бары или donut)
5. Слишком много текста, мало визуала
6. Итоги **generic** — не используют реальные данные

## Как будет выглядеть пример

### Слайд 1: Титульный
- Крупный заголовок «Отчёт за Июнь 2026»
- 3 KPI в ряд: Задач (18), Завершено (12), Прогресс (67%)
- Donut-диаграмма прогресса
- Подзаголовок «Команда разработки»

### Слайд 2: Динамика (НОВЫЙ)
- Столбчатая диаграмма: план vs факт по неделям (4 столбца)
- Ключевое число: «Скорость команды: 4.5 задач/неделю»
- Сравнение: «+15% к прошлому месяцу» или «−8%»

### Слайд 3: Статусы задач
- Donut-диаграмма сегментированная (каждый статус — цвет)
- Список справа: статус → кол-во → процент
- Цветовые бейджи

### Слайд 4: Бюджет часов
- 2 больших числа: План 120ч / Факт 104ч
- Прогресс-бар с зонами (зелёная / жёлтая / красная)
- Задачи с перерасходом — красным, сэкономившие — зелёным
- Итого: «Экономия 16ч (−13%)»

### Слайд 5: Завершённые задачи (ТОП-6)
- Компактный список, не карточки
- Каждая строка: № + название + статус-бейдж + часы
- Только ключевые задачи, не все 8-9

### Слайд 6: Задачи в работе (ТОП-6)
- Аналогично — компактный список
- Добавлен прогресс-бар каждой задачи (план/факт)

### Слайд 7: Риски и блокеры (НОВЫЙ)
- 2-3 карточки с конкретными задачами, которые просрочены/перерасход
- Красный акцент, конкретные цифры
- Не generic «есть риски», а «#35204 Тестирование API: план 16ч, факт 19ч, +3ч перерасход»

### Слайд 8: Итоги и следующие шаги
- 2 колонки: что сделали / что дальше
- Конкретные цифры, не generic текст
- CTA: «План на июль: 22 задачи, 156ч»
